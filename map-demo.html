World Shorelines

For a base layer on which to build, it'd be useful to simply display the world map. To do that we display the global shorlines. Shorelines of the world can be downloaded from https://www.ngdc.noaa.gov/mgg/shorelines/ Or also from https://www.soest.hawaii.edu/pwessel/gshhg/ (I think these two are actually the same)

You should get the ESRI shapefile version. The other formats (like NetCDF) seem to be even more complex.

The shorelines come at multiple resolutions and are split into different groupings. The README explains what the different files contain.

Conversion

Now we need to convert the shapefile to a GeoJSON. Unfortunately I'm not sure this can be done within the JVM. However there is a command line tool called ogr2org that will do this automatically for you. It's part of GDAL

https://gdal.org/programs/ogr2ogr.html

The tool docs don't directly mention GeoJSON for some reason, but you can find conversion one liners on the GeoJSON page

https://gdal.org/drivers/vector/geojson.html

For instance, I went into the shoreline subdirectory GSHHS_shp/c and ran 

ogr2ogr -f GeoJSON shoreline.json GSHHS_i_L1.shp

This gives you a GeoJSON of the shorelines of all the continents (this seem to also include small periheral islands as well). I loaded both in QGIS and at least to my eye they look the same.

We can then proceed in a few steps

  • construct a cropping polygon
  • use the intersect function in geo/jts to crop out a region of interest
  • save the cropped region to a GeoJSON
  • unfold the region's geometries into polygons. Sometimes intersect will cut a polygon into multiple polygons, and these will be grouped into a MultiPolygon object. You can also get GeometryCollection objects (unclear when). These too should be "unfolded". So that you are left with a long list of Polygons
  • you can directly extract the Polygon's coordinates
  • these coordinates can be directly used in thing/geom's SVG hiccup to make SVG polygons
  • there are some coordinate adjustments to rezero and flip the image
  • the SVG hiccup can be serialized and output as an SVG string (embedded in the HTML)
  (defn region-ranges
    [region]
    (let [{:keys [start-lat
                  start-lon
                  ended-lat
                  ended-lon]} region]
      [(Math/abs (- ended-lat
          start-lat))
       (Math/abs(- ended-lon
          start-lon))]))

  (defn crop-shoreline-to-region
    [shoreline
     region]
    (let [{:keys [start-lat
                  start-lon
                  ended-lat
                  ended-lon]} region
          [lat-range 
           lon-range] (region-ranges region)
          area-of-interest (polygon-wkt [[start-lon start-lat
                                          start-lon ended-lat
                                          ended-lon ended-lat
                                          ended-lon start-lat
                                          start-lon start-lat]])]
      (->> shoreline
           (map (fn [shoreline-poly]
                  (update shoreline-poly
                          :geometry
                          #(intersection %
                                         area-of-interest))))
           (filter #(not (.isEmpty (:geometry %)))))))

  (defn geojson-to-svg
    [geojson
     region
     attribs]
    (let [{:keys [start-lat
                  start-lon
                  ended-lat
                  ended-lon]} region
          [lat-range 
           lon-range] (region-ranges region)]
      (->> geojson
           (map :geometry)
           (reduce (fn [polygon-seq ;; extract polygons from multipolygons
                        next-geometry]
                     (println (.getGeometryType next-geometry))
                     (case (.getGeometryType next-geometry)
                       "Polygon"
                       (conj polygon-seq next-geometry)
                       "MultiPolygon"
                       (apply conj polygon-seq (polygons next-geometry))
                       "GeometryCollection" ;; This probably need to be recursive
                       (apply conj polygon-seq (geometries next-geometry))
                       ))
                   [])
           (map coordinates)
           (map coord-array) 
           (map (fn [poly]
                  (map #(vector (- (.getX %)    ;; zero to starting longitude
                                   start-lon)
                                (- lat-range ;; SVG's Y axis is inverted
                                   (- (.getY %) ;; zero to starting latitude
                                      start-lat)))
                       poly)))
           (map #(svg/line-strip %
                                 attribs))
           (apply svg/group {}))))

Rain Data

Now we want to read in rain data and overlay it. Rain data can be retrieved from TODO: ADD INFO

It comes in two primary formats, HDF5 and GeoTIFF.

HDF5
is a more meteorologically specific format. This is how the data arrives orginally
GeoTiff
is an additional format provided. It's an image file along with some metadata

GeoTIFF

We can read in TIFF files directly in Java as long as it's Java 9 or newer. On older version of Java you need to use a library/plugin that adds support.

The file provides precipitation data as a grayscale image of the whole globe. The higher the value of a pixel, the more mm of precipitation. In these particular files each pixel represents a 0.1 x 0.1 degree square

The image is 16bit and single channel. This is slightly unusual and Java only supports unsigned short (ie. 16bit positive numbers) values indirectly. If you start looking at a BufferedImage DataBuffer you will see what looks like an array of signed short values that range from -32767 to 32767. But we want them unsinged ranging from 0 to 65535. The easiest way is to use a helper function provided by Java, Short.toUnsignedInt, that will extract the value as if it were an unsigned short and copy it into an Int. We can then go through and copy out the whole buffer into a buffer of Ints before displaying them (we're only looking at a small region anyway)

Coordinate Chaos

  • A Java BufferedImage will naturally start with the [0,0] in the upper left-hand corner. This is the most common format for dealing with Images (and corresponds well to how you display things in a window - with the "root" always being the window's upper left-hand corner).
  • Geographic coordinates (such as in our GeoJSON files) come in [lat,long] pairs. The [0,0] point in the center (off the coast of Africa). Furthermore lat is the "height" on the map, while long is effectively the "width". This is the inverse of the usual ordering of [width,height].
  • Plots usually are drawn with [0,0] in the bottom left corner. (here I use a "heatmap" plot to display the rain "grid" data)
  (defn geotiff-to-matrix
    "read in a GeoTIFF image into a `thi.ng.ndarray` matrix
      This is later plotted in a heatmap with `thi.ng.geom`"
    [geotiff
     [lat-res
      lon-res]
     region]
    (let [;; A lat-lon region is a square from bottom left to upper right corner
          {:keys [start-lat
                  start-lon
                  ended-lat
                  ended-lon]} region 
          pix-per-deg-lat (/ 1.0
                             lat-res)
          pix-per-deg-lon (/ 1.0
                             lon-res)
          ;; An image region is top-left to bottom-right pixel
          start-pix-x (* (+ 180
                            start-lon)
                         pix-per-deg-lon)
          start-pix-y (* (- 90
                            ended-lat)
                         pix-per-deg-lat)
          width-pix (- (* (+ 180
                             ended-lon)
                          pix-per-deg-lon)
                       start-pix-x)
          height-pix (- (* (- 90
                              start-lat)
                           pix-per-deg-lat)
                        start-pix-y)
          [lat-range 
           lon-range] (region-ranges region)]
      (println " dimensions: " 
               start-pix-x
               start-pix-y
               height-pix
               width-pix
               (-> geotiff
                   java.io.File. 
                   javax.imageio.ImageIO/read
                   .getWidth)
               (-> geotiff
                   java.io.File. 
                   javax.imageio.ImageIO/read
                   .getHeight))
      (let [image-data (-> geotiff
                           java.io.File. 
                           javax.imageio.ImageIO/read
                           (.getData (java.awt.Rectangle. start-pix-x
                                                          start-pix-y
                                                          width-pix
                                                          height-pix))
                           .getDataBuffer
                           .getData)
            int-buffer (mapv #(Short/toUnsignedInt %)
                             image-data)
            max-value (apply max int-buffer)
            normalized-buffer (mapv #(* (/ 65535
                                           max-value)
                                        %)
                                    int-buffer)]
        (->> normalized-buffer
             (#(ndarray/ndarray :int32
                                %
                                [
                                 height-pix width-pix ]))
             #_(#(viz/matrix-2d lat-pix-width
                                lon-pix-width
                                %))
             #_alength
             #_(take 590)
             #_  (#(.getElem % 590))
             #_(into [])))))

  (def grey-scale-for-shorts
    "A grayscale linear color map - over all possible `unsigned short` values
      see: https://colorcet.com/userguide/index.html"
    (->> (range 1
                0
                (- (/ 1
                      65535)))
         (mapv #(col/hsla 0
                          0
                          %
                          1))))

  (defn geotiff-to-heatmap
    "Turns a GeoTIFF file into a `thi.ng.geom` heatmap plot"
    [geotiff
     [lat-res
      lon-res]
     [lat-range
      lon-range]
     region]
    (let [geotiff-matrix (geotiff-to-matrix geotiff
                                            [lat-res
                                             lon-res]
                                            region)]
      {:x-axis (viz/linear-axis
                {:domain [0 80]
                 :range [0 lon-range]
                 :visible false})
       :y-axis (viz/linear-axis
                {:domain      [0 50]
                 :range       [0 lat-range ]
                 :visible false})
       :data     [{:matrix        geotiff-matrix
                   :value-domain  [0 65535]
                   :palette       grey-scale-for-shorts
                   :palette-scale viz/linear-scale
                   :layout        viz/svg-heatmap}]}))


  (defn plot-rain
    "returns a `thi.ng.geom` SVG hiccup (using a heatmap) of a GeoTIFF rain"
    [rain-map-file
